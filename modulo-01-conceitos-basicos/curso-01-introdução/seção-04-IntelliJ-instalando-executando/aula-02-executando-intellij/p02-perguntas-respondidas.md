# Perguntas e Respostas — Executando no IntelliJ

1. O que é uma IDE?
   Um ambiente integrado para desenvolvimento.

2. O IntelliJ é uma IDE?
   Sim.

3. JDoodle é uma IDE?
   É uma IDE online.

4. IDE online substitui uma IDE local?
   Não.

5. Onde começa um programa Java?
   No método main.

6. Qual é a assinatura do main?
   public static void main(String[] args)

7. O que é args?
   Lista de argumentos do programa.

8. args pode estar em varargs?
   Sim.

9. System.out.println faz o quê?
   Imprime no console.

10. Ponto e vírgula é obrigatório?
    Sim.

11. javac faz o quê?
    Compila código Java.

12. java faz o quê?
    Executa código compilado.

13. javac gera o quê?
    Arquivos .class.

14. java executa o quê?
    Arquivos .class.

15. O IntelliJ compila automaticamente?
    Sim.

16. Run Configuration serve para quê?
    Configurar execução.

17. O que são program arguments?
    Parâmetros passados ao main.

18. O que é debug?
    Execução passo a passo.

19. O que é breakpoint?
    Ponto de parada no código.

20. Posso alterar variáveis no debug?
    Sim.

21. O IntelliJ mostra args?
    Sim.

22. External Libraries mostra o quê?
    JDK e dependências.

23. Gitignore serve para quê?
    Ignorar arquivos no Git.

24. O IntelliJ cria arquivos de config?
    Sim.

25. Posso rodar Java sem IDE?
    Sim.

26. O IntelliJ é obrigatório?
    Não.

27. args[0] representa o quê?
    Primeiro argumento.

28. args[1] representa o quê?
    Segundo argumento.

29. Posso passar quantos argumentos quiser?
    Sim.

30. java Main a b c passa quantos argumentos?
    Três.

31. O debug executa tudo de uma vez?
    Não.

32. Step Over faz o quê?
    Avança uma linha.

33. Step Out faz o quê?
    Sai do método atual.

34. Evaluate Expression serve para quê?
    Avaliar código em tempo real.

35. Posso testar código no debug?
    Sim.

36. IDE acelera desenvolvimento?
    Sim.

37. Terminal é importante?
    Sim.

38. Automação usa IDE?
    Não.

39. Maven e Gradle rodam no terminal?
    Sim.

40. Desenvolvedor deve dominar IDE e terminal?
    Sim.
