# Perguntas e Respostas — SDKMAN

## Conceituais

1. O que é o SDKMAN?
   Um gerenciador de SDKs e ferramentas de desenvolvimento.

2. Para que ele é usado?
   Para instalar e trocar versões de Java e outras ferramentas.

3. SDKMAN funciona no Windows?
   Sim, via WSL.

4. Ele funciona no Linux?
   Sim.

5. Ele funciona no macOS?
   Sim.

6. SDKMAN serve apenas para Java?
   Não.

7. Ele pode instalar Maven?
   Sim.

8. Ele pode instalar Gradle?
   Sim.

9. Ele pode instalar Spring Boot CLI?
   Sim.

10. Ele pode instalar GraalVM?
    Sim.

## Práticas

11. Como listar versões de Java?
    sdk list java

12. Como instalar uma versão?
    sdk install java <id>

13. Como ver a versão ativa?
    java -version

14. O SDKMAN configura o PATH?
    Sim.

15. O SDKMAN define a versão default?
    Sim.

16. Posso ter Java 8 e 21 juntos?
    Sim.

17. Posso trocar versões?
    Sim.

18. Preciso mexer em JAVA_HOME?
    Não.

19. O SDKMAN resolve conflitos de versão?
    Sim.

20. Ele baixa o JDK automaticamente?
    Sim.

## Estilo entrevista júnior

21. Por que usar SDKMAN?
    Para facilitar o gerenciamento de versões.

22. É melhor que instalar manualmente?
    Sim, para desenvolvimento.

23. É bom para servidores?
    Não tanto.

24. É bom para dev?
    Sim.

25. Ele evita erros de PATH?
    Sim.

26. Posso usar Java 21 no curso?
    Sim.

27. Posso usar Java 17?
    Sim, mas o curso usa 21.

28. O SDKMAN substitui o JDK?
    Não, ele gerencia JDKs.

29. Ele é seguro?
    Sim.

30. Ele funciona offline?
    Não.
