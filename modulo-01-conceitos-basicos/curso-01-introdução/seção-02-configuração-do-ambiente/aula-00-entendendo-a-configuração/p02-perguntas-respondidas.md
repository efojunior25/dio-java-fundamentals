# Perguntas e Respostas — Configuração do Ambiente Java

## Conceituais

1. O que é o ambiente Java?
   É o conjunto de ferramentas e configurações para programar em Java.

2. O que é o JDK?
   Java Development Kit.

3. O JDK é obrigatório?
   Sim, para programar em Java.

4. O que acontece sem o PATH configurado?
   Os comandos Java não funcionam.

5. O que é o OpenJDK?
   A versão open source do Java.

6. Oracle JDK é gratuito?
   Nem sempre, depende da versão e uso.

7. Amazon Corretto é compatível com Java?
   Sim.

8. Azul Zulu é uma versão oficial?
   É uma distribuição baseada no OpenJDK.

9. Posso usar qualquer JDK?
   Sim, todos seguem o mesmo padrão.

10. O que o comando `java -version` faz?
    Mostra a versão instalada do Java.

## Técnicas

11. Para que serve o `javac`?
    Compilar código Java.

12. O que é PATH?
    Variável que define onde o sistema procura executáveis.

13. Sem PATH posso rodar Java?
    Somente informando o caminho completo.

14. O que é JVM?
    A máquina virtual do Java.

15. JDK contém a JVM?
    Sim.

16. Posso ter mais de um Java instalado?
    Sim.

17. Como trocar versões facilmente?
    Usando SDKMAN.

18. SDKMAN funciona no Windows?
    Não oficialmente.

19. SDKMAN instala apenas Java?
    Não, várias SDKs.

20. Como verificar se tudo funciona?
    Executando `java -version` e `javac -version`.

## Estilo entrevista júnior

21. Por que preciso instalar o JDK?
    Para compilar e executar Java.

22. Qual JDK devo usar?
    OpenJDK ou algum derivado.

23. O que acontece se não configurar o PATH?
    O Java não roda.

24. O que é uma distribuição?
    Uma forma empacotada do JDK.

25. Java depende de sistema operacional?
    Não, por causa da JVM.

26. Posso usar Java no Linux?
    Sim.

27. Posso usar Java no macOS?
    Sim.

28. Posso usar Java no Windows?
    Sim.

29. Como sei que Java está instalado?
    Com `java -version`.

30. Por que SDKMAN é útil?
    Porque facilita trocar versões.
