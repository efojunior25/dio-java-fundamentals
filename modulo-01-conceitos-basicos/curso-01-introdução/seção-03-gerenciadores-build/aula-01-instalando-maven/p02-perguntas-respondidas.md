# Perguntas e Respostas — Instalando o Maven

1. O que é o Maven?
   Uma ferramenta de build para projetos Java.

2. Por que o Java precisa estar instalado antes?
   Porque o Maven roda sobre a JVM.

3. Qual versão de Java o professor usa?
   Java 21.

4. O Maven é instalado via instalador?
   Não, via zip extraído.

5. Onde o professor colocou o Maven?
   Inicialmente em C:\Maven.

6. O que é MAVEN_HOME?
   Variável que aponta para a pasta do Maven.

7. O que é PATH?
   Lista de diretórios onde o sistema procura comandos.

8. O que deve ser adicionado ao PATH?
   %MAVEN_HOME%\bin.

9. Qual comando testa o Maven?
   mvn -version.

10. Por que o comando não funcionou de início?
    Porque o PATH ainda não estava configurado.

11. Por que funcionou como administrador?
    Por causa de permissões.

12. Qual foi o problema real?
    A pasta C:\ tinha restrições.

13. Qual foi a solução?
    Mover o Maven para a pasta do usuário.

14. Maven funciona sem Java?
    Não.

15. Maven precisa de JAVA_HOME?
    Sim.

16. O Maven foi instalado corretamente?
    Sim.

17. O erro foi de configuração?
    Não, foi de permissão.

18. O Windows pode bloquear pastas?
    Sim.

19. Maven é indicado para servidores?
    Sim.

20. Maven é flexível para dev?
    Menos que SDKMAN.

21. O professor prefere SDKMAN para dev?
    Sim.

22. Posso instalar Maven manualmente?
    Sim.

23. O que acontece se MAVEN_HOME estiver errado?
    Maven não funciona.

24. PATH pode ter conflitos?
    Sim.

25. mvn é o executável do Maven?
    Sim.

26. A pasta bin contém o quê?
    Os executáveis.

27. Posso ter múltiplos Mavens?
    Sim, mas gera conflito.

28. Maven baixa dependências?
    Sim.

29. Maven compila projetos?
    Sim.

30. Maven empacota projetos?
    Sim.

31. mvn -version mostra o quê?
    A versão do Maven.

32. O Maven é open source?
    Sim.

33. Ele roda no Windows?
    Sim.

34. Ele depende do Java?
    Sim.

35. O problema do vídeo foi didático?
    Sim, mostrou um erro real.
