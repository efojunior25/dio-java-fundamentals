# Perguntas e Respostas — Gerenciadores de Build

1. O que é build?  
   Processo de transformar código em software executável.

2. O que um build faz no Java?  
   Compila, baixa dependências, roda testes e empacota.

3. O que é um gerenciador de build?  
   Ferramenta que automatiza o build.

4. Por que não fazer build manualmente?  
   Porque é lento e propenso a erros.

5. Quais são os principais gerenciadores Java?  
   Maven e Gradle.

6. O Maven usa o quê?  
   XML.

7. O Gradle usa o quê?  
   Scripts Groovy ou Kotlin.

8. O que é convenção sobre configuração?  
   Usar padrões em vez de configurações manuais.

9. O que é build.gradle?  
   Arquivo principal do Gradle.

10. O que é um .jar?  
    Arquivo empacotado de uma aplicação Java.

11. O Maven é antigo?  
    Sim, mas muito usado.

12. O Gradle é moderno?  
    Sim.

13. Maven é flexível?  
    Menos que o Gradle.

14. O Gradle é mais poderoso?  
    Sim.

15. O que é MAVEN_HOME?  
    Variável que aponta para o Maven.

16. O que é GRADLE_HOME?  
    Variável que aponta para o Gradle.

17. O que é PATH?  
    Lista de diretórios onde o sistema procura comandos.

18. Por que o bin precisa estar no PATH?  
    Para rodar mvn e gradle.

19. O que causa “mvn não é reconhecido”?  
    PATH mal configurado.

20. Variáveis de usuário afetam quem?  
    Somente o usuário.

21. Variáveis do sistema afetam quem?  
    Todos os usuários.

22. Posso ter conflito entre elas?  
    Sim.

23. O que acontece se houver dois Mavens?  
    O primeiro no PATH é usado.

24. Maven baixa dependências?  
    Sim.

25. Gradle baixa dependências?  
    Sim.

26. Gerenciadores executam testes?  
    Sim.

27. Eles empacotam projetos?  
    Sim.

28. Maven gera .jar?  
    Sim.

29. Gradle gera .jar?  
    Sim.

30. Posso usar Maven e Gradle juntos?  
    Sim, em projetos diferentes.

31. Um build pode falhar?  
    Sim.

32. O que é build automatizado?  
    Build executado por ferramenta.

33. Gerenciadores evitam erros humanos?  
    Sim.

34. Maven é corporativo?  
    Sim.

35. Gradle é popular em projetos modernos?  
    Sim.
