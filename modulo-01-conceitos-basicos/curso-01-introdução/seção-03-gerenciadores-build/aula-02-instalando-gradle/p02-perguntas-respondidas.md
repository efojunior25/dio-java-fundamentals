# Perguntas e Respostas — Instalando o Gradle

1. O que é o Gradle?
   Uma ferramenta de build para projetos Java e JVM.

2. O professor prefere instalar o Gradle manualmente?
   Não, ele prefere o SDKMAN.

3. Por que ele mostrou a instalação manual?
   Para ensinar como funciona o ambiente.

4. Onde o Gradle é baixado?
   No site oficial do Gradle.

5. Qual versão foi usada?
   Gradle 8.13.

6. O que é GRADLE_HOME?
   Variável que aponta para a pasta do Gradle.

7. O que deve ser adicionado ao PATH?
   %GRADLE_HOME%\bin.

8. Qual comando testa o Gradle?
   gradle -v.

9. O que acontece antes da configuração?
   O comando não é reconhecido.

10. Por que é preciso extrair o ZIP?
    Para acessar os binários.

11. O Gradle funciona sem Java?
    Não.

12. Ele depende da JVM?
    Sim.

13. A versão binária é suficiente?
    Sim.

14. O Gradle mostra versão do Kotlin?
    Sim.

15. O Gradle mostra versão do Groovy?
    Sim.

16. O que o gradle -v exibe?
    Informações da ferramenta.

17. Posso ter múltiplos Gradles?
    Sim.

18. PATH pode causar conflitos?
    Sim.

19. O Gradle precisa de GRADLE_HOME?
    Sim.

24. Gradle executa builds?

28. Ele pode usar Kotlin?

    Sim.

    Sim.
